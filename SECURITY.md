# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

We take security seriously. If you discover a vulnerability, report it privately using
[GitHub Security Advisories](https://github.com/Mikeus/next-service-pages/security/advisories/new).

Include as much detail as possible:

- Description of the vulnerability and potential impact
- Steps to reproduce
- Affected versions or components
- Any suggested fix or mitigation (optional)

## Response Timeline

| Stage       | Target                                      |
| ----------- | ------------------------------------------- |
| Acknowledge | Within **48 hours** of report               |
| Assessment  | Within **5 business days**                  |
| Patch       | Within **14 days** for confirmed issues     |
| Disclosure  | Coordinated after a fix is ready to release |

We will keep you informed of progress and credit reporters in the advisory unless you
prefer to remain anonymous.

## Scope

Reports are in scope when they affect:

- This repository's application code, API routes, and configuration
- Dependencies with a direct, exploitable impact on users of this starter kit

Out of scope: social engineering, physical attacks, third-party services you configure
separately (OpenAI, Anthropic, hosting providers), and issues in forked deployments
with custom modifications.
